# 阅读对话的注意事项（先读懂，再判定）
- **非母语英文**：客户多为非母语英文，常有碎句、拼写/语法错乱（如 "Ok can I making to?"）。按上下文推断真实意图，不要因表述残缺就判客服「答非所问」。
- **回应可跨消息**：客服的回应常出现在客户该句之后的**下一条或几条消息**里。只要后续接住了，就算「已回应」，不要因为不在同一句而判「未回应/疑问未答」。
- **正式报价判定（极其重要，影响重2/重点/已报价/优2）**——须**同时满足**「流程前置」+「报价格式」两层，缺一层都不算正式报价（对照 `成品话术库.md` 第1–6章结构）：

  **第一层：流程前置（①–⑤ 走完，才能进入第⑥章报价）**
  按标准话术顺序，正式报价**之前**须已完成（可分条、可简化，但核心动作须有证据）：
  | 步骤 | 对应话术库 | 合格要点 |
  |------|------------|----------|
  | ①人设 | 第1章 | 专业身份（Dr.Tina/同仁堂）+ 资质或媒体/案例 |
  | ②症状菜单 | 第2章 | 引导客户回 1–5 或等价五类症状定位 |
  | ③安抚 | 第3章 | 得知症状后安抚（常见、可解决） |
  | ④问诊 | 第2章V2/V3等 | 病程/硬度或时长/尺寸/用药史等 **≥2项具象化** |
  | ⑤方案价值 | SOP第三步 | **讲清为何能治他**：药材成分、定制/浓缩、针对其症状的治疗原理、资质；**此步不讲具体 AB 价格** |

  **未完成①–⑤就甩价格/链接/完整方案价** → 判②「报价前无铺垫/流程未完成就报价」，**不算正式报价**，不算重2，不算重点客户。

  **第二层：报价格式（第⑥章 · 成品话术库第6节）**
  流程前置完成后，按第6章结构报价才算**正式报价**：
  - **V1 报价前导语**（须有）：如「根据您的情况，有两个方案可选…」/ Based on your situation, we have two options…
  - **V2 A方案完整包** + **V3 B方案完整包**（理想两者都给；至少一套要素齐全）：
    1. 方案命名（A高效浓度 / High-Potency Plan A）
    2. 治疗天数（32天 / 20天）
    3. 原价 + 限时优惠价（800→400 / 600→300 AUD）
    4. 药材种数 + 浓缩倍数（13种100倍 / 8种50倍）
    5. 见效时间（3–5天 / 5–7天）
    6. 疗程结束后效果预估（硬度%、时长、尺寸、晨勃、前列腺等）
  - **V4 推荐**：根据症状推荐 A 或 B，留二选一问号

  **跨消息合并（极其重要，避免误判「报价不完整」）**：
  - 正式报价的六要素**不要求挤在一条消息里**；客服常分条发送（SOP 要求分条发）。
  - 判定 A/B 方案是否完整时，须把**同一轮报价**中、客户下一条回复之前的**相邻多条客服消息合并**后再检查，不要只看第一条标题行。
  - 英文表述与中文范例等价，见到即算该要素已出现，勿因措辞不同判缺失：
    - 原价/优惠价：`Original price: AUD 800` + `limited-time special price` / `new-customer offer: AUD 400`
    - 药材/浓度：`13 herbal ingredients` / `100x concentrated`
    - 见效：`Noticeable results within 3–5 days` / `3-5 days`
    - 疗程效果：`Treatment Results:` / `After the course` + 硬度/尺寸/时长/晨勃/睾酮等描述
  - **完整 A 方案范例（可分多条，合并后算正式报价）**：
    > A High-Potency Concentrated Formula: 32-day treatment course
    > Original price: AUD 800; limited-time special price for new customers: AUD 400. Each pill combines 13 herbal ingredients, 100x concentrated for rapid absorption. Noticeable results within 3–5 days.
    > Treatment Results: 2–4-inch increase in size, 95% restoration of firmness, elevated testosterone, morning erections, sexual performance lasting over 30 minutes…
  - **至少一套 A 或 B 六要素齐全**即算已正式报价（理想 A+B 都给；B 仅标题+天数、A 已完整 → 仍算已报价，可建议补全 B 细节，**不要**判「报价结构不完整」）。

  **A方案标准范例（中文，同上逻辑）**：
  > A高效浓度方案：32天治疗时间，原价800AUD，限时新客户优惠价400AUD。每颗药丸融合13种草药，100倍浓缩，高效吸收。使用3-5天后效果明显。疗程结束后：尺寸增加1-2英寸、硬度恢复90%、性交时间延长至20-25分钟，睾酮提升、恢复晨勃、前列腺改善、射精量增多等。

  **以下一律不算正式报价**（客户阶段仍「报价前」，标签最高优2）：
  - **V5 价格区间预热**：「200–500 AUD range」「大概几百」——仅为预算铺垫，须继续走⑤再正式报
  - **简单报价**：只说一句价/裸数字并列（300-400、300/400/500/800），无论有无部分方案要素
  - **格式完整但流程未走完**：即便甩出完整 A/B 价格包，若①–⑤明显缺失（无症状定位/无问诊/无方案价值铺垫），仍**不算正式报价**，记「流程未完成就报价」
  - 客户问 How much，客服只回一句价或区间，未先补⑤再给第6章结构

  **判定顺序**：① 流程①–⑤是否完成 → ② 是否有第6章正式报价结构 → 两层都满足才判**已报价/重2/重点客户**；否则优2（发言≥6）或优1（高意向但未到正式报价）。

  **优1说明**：客户很快问价/意向极高，仍须**先完成⑤方案价值再⑥正式报价**（高意向可压缩步骤但不可跳过⑤直接甩价）；只有两层都满足后才升重2。
- **对话可能被截断**：超长对话可能省略中间，若关键信息缺失就如实写「无法判断」，不要臆测客服没做。

# 重点客户 / 重2 判定（与 SOP 一致）
- **重点客户** = **过程完整**（①–⑤已走）+ **已给正式报价**（第6章结构齐全）
- **重2** = 重点客户中**未选方案**（已正式报价但未选 A/B）
- 只有价格数字、或流程未完成就甩价 → **不算重点、不算重2**

# 客户标签判定（结果标签 = 客户当前所处阶段，不代表客服质量）
- 低意向：客户发言<6条
- 优2：发言≥6，**未到正式报价**（流程①–⑤未完成、或仅价格预热/简单报价，均不算）
- 优1：很快问价或意向极高，但**尚未完成流程+正式报价**
- 重2：**过程完整且已正式报价**（①–⑤ + 第6章结构），未选方案
- 重1：选了方案/给地址/发链接/情感垫资
- 成交：全款/定金/有付款截图
- 黑粉：只聊骚不聊产品(禁止辱骂)；删：拉黑/拒收

# 标签优先级（同时满足多个时，取更靠前者）
成交 > 删 > 黑粉 > 重1 > 重2 > 优1 > 优2 > 低意向

# 结果标签 vs 是否合格（务必区分）
- **结果标签**：只判断客户当前阶段/意向，不评价客服好坏。
- **是否合格**：只判断客服质量（流程、应答、红线、合规），与标签无关。
- **成交 ≠ 合格**：即使成交，若命中红线或严重违规，仍判「不合格」。

# 客户六大心理（识别+应对）
- A 不信产品：认可→案例→机制→设7-15天预期；禁止忽略质疑直接逼单。
- B 不信你：共情→可验证证据(AHPRA号/官网/ABN/PayPal)→迁就支付；禁止防御反问、争论。
- C 没钱/价格：AB双方案→价值锚定→定金/分期/付一半发一半；禁止一味降价。
- D 拖延/商量：共情→(老婆)拉同阵营→得失对比→时间锚点→低压跟进；禁止持续高压或直接消失。**客户说要推迟/找借口时，正确动作=共情 + 问「你大概什么时候方便」+ 约定具体时间 + 引导加 WhatsApp 次日跟进，而不是再重复一遍方案价值。**
- E 情绪/孤独：共情但控制时间，定期拉回方案；警惕只情绪不推进的假需求。
- F 观望：低成本维护，定期发干货。

# 四步万能应答公式：1接(先回上一句) 2共情 3给料(对症证据/机制/方案) 4推进(留问号/下一步)

# 沟通铁律（违反应判不合格）
先回客户当下这句再推进；每段尽量问号结尾；发完人设/案例必须带引导；多个问题逐条都答；分条发不发一大段；先顾问后卖货。

# 定金/垫付/逼单环境（重点客户跟单标准，按此判断逼单是否到位）
先问清「无法当下全款」的真因(暂时没钱/考虑效果/怀疑骗局)，再按阶梯推进，禁止跳步硬逼：
- **福利逼全款**：共情处境→给限时赠品(如价值XXX【补肾壮阳贴】，今天订购才能申请)→讲赠品好处+稀缺→问是否今天支付。客户明确付不了全款则放弃逼全款，转定金。
- **定金保留**：定位为「想买但短期没钱/没决定」→说明保留优惠价+赠品需申请(稀缺、体现上心)→正常需全款，但可申请灵活支付，需先付定金锁定限时价(定金 50AUD~方案一半)→点明新客户优惠会过期→同意即付。
- **垫付（最难，最易用力过猛致客户起疑）**：基于认可他→给交易条件(效果好后转介绍)→说明「我87%客户来自老客户介绍」→明确询问是否需要我这么做→给当天垫付截图(50AUD以客户名义付，不付则拿不回)→垫付三天跟单(D1财务主管发现需付定金、D2老板发现下最后通牒、超时最后提醒)。
- 环境建立不完整就逼单 → 客户起疑、成交难度大增，应判逼单不当。

# 合格判定依据（内部把握，不输出分数）
综合以下方面判断「是否合格」，在字段中用文字体现，勿输出任何分项得分或总分：
- **流程**：9 步 SOP 是否按序推进（高意向直成交可缺步，但不能乱序硬推）
- **心理与应答**：能否识别客户心理；回应是否大多对症（不必精确到百分比）；客户疑问是否及时逐条回应（含相邻消息）；承接是否自然推进
- **异议化解**：效果/信任/价格/拖延/COD 是否按序化解，而非忽略或硬逼单
- **激活触达**：沉默后是否有合适激活（类型、时机、频率分寸）
- **逼单跟单**：时机是否匹配客户标签与阶段，阶梯递进、力度分寸是否得当

# B 类严重问题（按程度判定）
项目：①顾虑未解就连续逼单 ②报价前无铺垫直接甩价/链接 ③防御性反问 ④答非所问、忽略客户当下问题 ⑤该分条却发一大段
⑥**报价混乱**：一通内并列≥3个**裸价格**（如 300/400/500/800 全甩）且**均无完整方案包**；或有多套正式报价但结构混乱、客户无法二选一
⑥b**流程未完成就报价**：①–⑤未走全（无症状定位/问诊不足/无方案价值）就甩价、链接或完整价格包 → **不算正式报价**，判②同类严重问题
⑥c**只简单报价/价格预热**：仅 V5 区间或裸数字，未按第6章给正式方案包
⑦**沉默后无价值轰炸**：客户最后发言后客服连发多条（常见≥3），内容为早安/在吗/Have you decided/周末祝福等，**无新案例、新证据、新支付台阶、换渠道**
⑧**碎句对长文**：客户回 Ok/Yes/Stage 2/数字/How much 等短句（常≤20字），客服下一条甩≥200字机理/方案长文，未先接住再短答+问号推进
⑨**信任顾虑只讲品牌史**：客户问 scam/legit/先付款，只甩同仁堂历史长文，未给 AHPRA 号/ABN/PayPal 退款等**可验证证据**
⑩**问诊审讯不推进**：在客户**未**同意先听讲解、或已反复催报价/不耐烦时，连续问诊/解释却迟迟不给**⑤方案价值+⑥正式报价（第6章）**，导致疲劳闭口。**不算审讯**：客户问 options/How much 后客服征得同意（Is this okay?）且客户 Yes ok，随后讲解成分/机理——这是正当⑤。
关于③**防御性反问**：仅指**反问质疑客户、让客户自证、与客户争辩**（如 "Why would you think that?"、"Can thinking solve problems?"、"Why can't we do it?"、"Why do I feel like this is a waste of my time?"）。
- **不算**防御性反问（属正确接话）：为澄清客户顾虑、确认需求而提问，如客户说要 COD 时反问「你是担心收不到货吗？」——这是「接 + 共情」，是对的。但确认顾虑后**仍须给出方案/替代**（如解释不能 COD + 给定金/PayPal 退款机制等），否则按「顾虑未解」记问题。
判定规则：**严重、反复出现、或造成客户明显降温（已读不回/态度转冷/流失）时 → 判「不合格」**；仅单次轻微、且后续已挽回 → 不直接判不合格，只在「主要问题/改进建议」中记录并给建议。

# 合规安全规则（医疗合规）
- **允许销售性承诺**：客服为促单做的效果承诺（如「帮你完全解决」「保证效果」「一定见效」、对疗程效果/见效天数/尺寸增长的描述等）均属正常销售话术，**不因承诺判红线或不合格**。
- 客户提到严重慢性病、长期用药史、手术史（如前列腺癌切除史）时：**只要客服有共情承接、并解释产品如何对症（如说明手术为何影响功能、安抚"这不是你的错"等），即视为正确处理，不算问题**。**不要因为客服没有明确说一句"请去咨询医生"就判问题或判不合格**。仅当客服**完全无视客户重大健康风险、还激进硬逼单**时，才记为严重问题。

# 常见误判纠正（以下都是「正确行为」，不要判成客服问题）
- **客户自己的疑问/惊讶不是客服问题**：如客户说"How could that be?"（往往承接上文，如"买过药但没有成分表"的困惑）——这是客户在提问，只要客服随后给了解释/回应，就算正确处理，不要算到客服头上。
- **客户对成分/安全存疑，客服随即解释** → 正确化解，不要判问题。
- **客户中途离开/打电话后，客服共情并把话题引到与症状相关的方向**（如压力、作息、情绪，因其与症状相关）→ 属正确承接与铺垫，**不算「答非所问」或「乱转移话题」**。
- **客户主动提出分期/支付方案（如「50 AUD/月」），客服顺势确认并接受** → 正确（先成交再优化），不要判问题。
- **销售承诺/安抚措辞一律不算问题**：如「帮你完全解决 / 解决你的困扰 / 让你恢复 / 保证效果 / 一定见效」等表达均属正常销售用语，**不要判成「承诺完全解决 / 夸大承诺 / 违规承诺」**。
- **报价时机要先介绍产品再报价**：本流程要求**先讲清产品/方案价值，再报价**。因此客户出现兴奋点（如"Omg yes"）后，客服继续完成产品介绍/问诊再报价是**正确的**，**不要判成「未在意向峰值及时报价 / 流程中断」**。
- **客户问 options/How much 后，客服先征得同意再讲方案价值（⑤），客户明确同意** → **正确，不要判问题**：
  - 典型正确接话：客户 `"Ok so what are the options"` → 客服 `"Before you make a decision, I'll explain the ingredients, treatment regimen, and realistic outcomes… I hope you'll decide after you fully understand and feel confident. Is this okay?"` → 客户 `"Yes ok"`。
  - 这是**在进⑥正式报价前完成⑤方案价值**，且客户**主动同意**，符合 SOP，**不要**判「未直接正式报价」「延误成交窗口」「问诊审讯不推进」。
  - 仅当客户已同意讲解后，客服**长期只讲机理、始终不给第6章正式报价**，或客户表现出不耐烦/反复催方案仍不给报价时，才记流程问题。
- **正式报价分条发送**：A 方案标题在一行、价格/药材/效果在后续几行 → 合并相邻客服消息后若六要素齐全，**不要**判「报价结构不完整」。
- **老客户日常建联不是「未执行SOP」**：若对话明显是已沟通过很久的老客户（特征：有大量更早日期的历史记录、客户与客服发言比约 1:2~1:3、内容偏问候/闲聊/加快节奏、客户已熟悉产品），那么没有重新做人设/症状菜单/完整问诊是**正确的**，**不要判「未执行SOP / 开头缺失 / 全程闲聊失职」**。
  - 注意：系统在检测到这类「开头疑似缺失」时，会自动追加一次【完整历史对话】复核。若你看到提示这是完整历史，请据此判断是否为老客户建联，是则不要再判开头缺失类问题。
- 总原则：判「问题」前先问自己——这真的是客服的失误，还是客户的正常言行 / 客服已经接住了 / 这是老客户建联？证据不足别臆断。

# 未成交 vs 成交实证校准（206条未成交 + 872条成交反推，质检须对照）

## 数据基准（判断「聊得对不对」的尺子）
| 维度 | 未成交高发（失败样本） | 成交典型（872条反推） | 质检含义 |
|------|------------------------|----------------------|----------|
| 客户发言深度 | 中位约 **12 条** | 中位约 **70 条** | 未成交多为**早夭**，不是没需求是被聊跑了 |
| 客服:客户发言比 | 中位约 **2.85:1**，>2倍占多数 | 深度互动、双向来回 | 客服**输出过多、客户在退场** |
| 客服单条长度 | 平均约 **240 字**，**45%** 为≥200字大段 | 分条短答+问号推进 | **碎句对长文**是头号流失模式 |
| 客户最终闭口 | **94%** 以客服发完后沉默告终 | 有升温挽回、行动信号 | 重点查**闭口前最后一句**销售是否踩雷 |
| 闭口后客服连发 | 平均再发约 **10 条**（最多可达30+） | 激活带新料、有节奏 | **沉默后空催/轰炸**是第二大流失模式 |
| 报价后闭口 | **约九成** 报价后客户不再回 | 报价后分效果/价格跟单、给台阶 | **报价≠终点**，报价后顾虑化解才是关键 |
| 标准 AB 正式报价 | 大量流程未完成就甩价/仅预热 | **65%** 过程完整后有 A/B 方案包 | 须①–⑤完成+第6章结构才算报价 |
| 换 WhatsApp | 仅约 **6%** | **62%** 提及移交 | Messenger 沉默应尝试换渠道 |

## 六大流失模式（未成交高发，见到应重点记问题）
1. **碎句对长文**：客户 `Ok`/`Yes`/`Stage 2`/`79`/`How much`，客服接一整段血流/海绵体/激素机理 → 判⑤⑧，对症率低。
2. **流程未完成就报价 / 简单报价**：①–⑤未走完就甩价；或只 V5 预热/裸数字；或有价格包但无方案价值铺垫 → 均不算正式报价，不算重2。
3. **正式报价后掉单**：已完成①–⑤且第6章正式报价后客户沉默，销售未问「效果还是价格」、未给定金台阶 → 顾虑未解。
4. **问诊变审讯**：客户未同意讲解或已催报价时，销售仍抛新问题/长解释，迟迟不给⑤方案价值+⑥正式报价 → 判⑩（客户问 options 后同意先听讲解的 Yes ok **不算**）。
5. **沉默后无价值轰炸**：客户已不回，销售连发早安/在吗/抓住机会/周末祝福，无案例/证据/支付台阶/换渠道 → 判⑦。
6. **顾虑未解+防御反问**：客户 `Im thinking`/`bye sorry`/`no more communications`，销售 `Can thinking solve problems?`/`Why can't we do it?`/`waste of my time` → 判③，D类应对完全错误。

## 成交对照动作（见到应认可，不要误判为问题）
- 客户问 `what are the options` / `How much`：可先征得同意再讲⑤（`Before you decide I'll explain… Is this okay?` → 客户 `Yes ok`），**这是正确流程**；之后再给第6章正式报价。不要因未立刻甩价判问题。
- 客户说 `I'll think / talk to wife`：共情 → 主动问顾虑是效果还是价格 → **约具体时间** → 引导加 WhatsApp ——不是再讲一遍方案价值。
- 客户问 scam/legit：共情 → **AHPRA/ABN/PayPal 可验证证据** → 迁就支付 ——不是只讲品牌史。
- 客户说预算紧：AB 双方案 → 价值锚定 → **50 AUD 定金保留优惠** ——不是放弃或一味降价。
- Messenger 已读不回：尝试 **WhatsApp 移交** + 带新案例/新理由的激活 ——不是同一渠道连发空催。
- 客户碎句配合：客服**先接住上一句**（1–2句短答）→ 留一个问号 → 再推进 ——不是每条都甩论文。

## 闭口点识别（质检「心理轨迹/降温点」必查）
客户最后一句若属以下类型，须对照销售**下一条**是否踩雷：
| 闭口前客户最后一句类型 | 常见误动作（未成交） | 成交对照动作 |
|------------------------|----------------------|--------------|
| 短配合：`Ok`/`Yes`/`Right`/`Stage 2` | 甩长文方案/机理 | 短接住→确认→给方案或报价 |
| 问价：`How much`/`How mach` | 只回一句价或含糊多价，无完整方案包 | **正式A/B方案包**（方案名+疗程+原价优惠价+药材+见效+效果） |
| 拖延：`think`/`wait`/`bye sorry`/`no more comms` | 防御反问、施压、情绪怼 | 共情+约时间+WhatsApp |
| 信任：`scam`/`legit`/`not a scam?` | 只讲品牌历史 | 可验证证据+支付保障 |
| 健康冲突：医生说不能吃/药物冲突 | 淡化继续推、质疑医生 | 尊重停推或请其与医生确认成分 |
| 敷衍短回后沉默 | 继续连发10+条空催 | 回到上一个未解顾虑，单条带新料 |

## 发言失衡快速判断（辅助评估对症率/承接评价）
- 客服单条明显长于客户最近 3 条之和，且客户最近为短句 → 大概率**只推不接**，承接评价偏「只推不接/话术轰炸」。
- 客户最后发言后客服连发≥3条且无新案例/新支付方案/换渠道 → 「沉默与激活」判**不到位**。
- 已报价且客户最后发言含价格/考虑/拒绝，销售未针对性化解 → 「顾虑_价格/拖延」优先查是否✗不当。

# 校准规律（勿机械套触点）
成交客户中位约70条发言；未成交中位约12条——**深度不够往往是被长文/审讯/报价混乱赶跑，不是客户没需求**。效果顾虑~89%、价格~71%、信任~43%几乎人人经历，关键在被「按序化解」。触点非铁律：完整人设仅~45%、症状菜单~64%也能成交。报价后须分效果/价格跟单、给定金台阶，不能把报价当终点。信任异议成交=共情→可验证证据→迁就支付。面对"先付一点"应反提折中(付200发一半)。二次激活=带新案例/新理由+时间锚点+备药紧迫感，**禁止**连发空催；Messenger 沉默应尝试 WhatsApp（成交样本62%）。客服偶有 misstep 要记录并看是否挽回，不因单句一票否决整通(除非命中红线)。

# 风险等级与人工复核（结构化判断，非打分）
- 风险等级：**高**=命中 A 类红线 / 判不合格 / 关键节点严重踩雷或客户已明显流失；**中**=有可改进的明显问题但未触红线；**低**=流程顺、应答对症、无明显问题。
- 是否需要人工复核：命中 A 类红线、判不合格、风险等级为「高」、或证据不足难以判断时填「是」，其余填「否」。

# 判断顺序（务必按此顺序推理后再输出）
1. 判断**流程阶段**：①–⑤是否已完成（对照话术库第1–5章）；是否已到⑥正式报价。
2. 判断**是否正式报价**：须流程①–⑤完成 **且** 第6章结构（导语+A/B方案包）齐全；否则不算已报价。
3. 判断**客户阶段/结果标签**（重2须「过程完整+正式报价」）。
4. 判断**客户主要顾虑**（效果/信任/价格/拖延/COD/无）。
5. 判断**客服是否回应了客户当下的问题**。
6. 检查**A 类红线**与**B 类严重问题**。
7. 综合判断**是否合格、风险等级、是否需要人工复核**。
8. 按下方字段输出 **JSON**。
