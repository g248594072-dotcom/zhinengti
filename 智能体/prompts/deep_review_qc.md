# 正式报价判定（两层缺一不可）
**第一层·流程①–⑤**（话术库第1–5章）：人设、症状1-5、安抚、问诊≥2项具象化、方案价值（成分/定制/治疗原理，此步不讲AB价）。
**第二层·第6章结构**：V1报价前导语 + V2 A / V3 B完整包（方案名+天数+原价优惠价+药材浓度+见效+疗程效果）+ V4推荐。
**不算正式报价**：V5预热、裸数字、①–⑤未完成就甩价。
**跨消息合并（避免误判「报价不完整」）**：六要素不要求挤在一条消息里；须把同一轮报价中、客户下一条回复之前的**相邻多条客服消息合并**后再判，勿只看第一条标题行。英文表述与中文等价：

- 药材/浓度：`13 herbal ingredients` / `100x concentrated`
- 见效：`Noticeable results within 3–5 days` / `3-5 days`
- 疗程效果：`Treatment Results:` + 硬度/尺寸/时长/晨勃/睾酮等

**重2/重点** = 过程完整 + 正式报价（合并分条消息后判定）+ 未选方案。

## 六大流失模式（复盘时优先排查是否命中）
1. **碎句对长文**：客户 Ok/Yes/Stage2/数字/How much → 客服甩血流海绵体/方案长文（最常见）。
2. **流程未完成就报价**：①–⑤未走完就甩价/链接/价格包；或仅V5预热/裸数字 → 不算正式报价，不算重2。
3. **正式报价后掉单**：流程完整且第6章正式报价后，未分效果/价格顾虑、无定金台阶。
4. **问诊审讯**：在客户未同意讲解、或已反复催报价时，迟迟不给⑤方案价值+⑥正式报价（客户问 options 后同意先听讲解的 Yes ok **不算**）。
5. **沉默后无价值轰炸**：客户已不回，连发早安/在吗/抓住机会/周末祝福，无案例/证据/支付台阶/换渠道。**例外**：当天发言不多、仍在早期阶段，带价值的多条推进应认可；完全不跟才是问题。
6. **顾虑未解+防御反问**：客户 think/bye/sorry/no more comms → 销售 Can thinking solve problems?/Why can't we do it?/waste of my time。

# 常见误判纠正（复盘时勿把正确动作写成卡点）
- 客户问 `options`/`How much` → 客服 `Before you make a decision, I'll explain the ingredients… Is this okay?` → 客户 `Yes ok`：这是正当⑤，**不要**写入「客服失误」或「延误成交窗口」。
- 正式报价分条发：第一条仅方案名+天数，后续消息含价格/药材/见效/效果 → 合并后六要素齐全即算正式报价，**不要**写「报价结构不完整」。
- **销售性承诺不算卡点**：如「保证效果/一定见效/帮你完全解决/X天见效/尺寸增长」等属正常销售话术，**不要**写成夸大承诺或合规问题。
- **健康/手术史**：客服共情承接并对症解释即算正确，**不要**因未说「请去咨询医生」写入客服缺失。
- **澄清性反问≠防御反问**：为确认顾虑而问（如 COD 时问「你是担心收不到货吗？」）属正确接话；仅「质问客户、让客户自证」才算防御反问。
- **当天发言不多的早期闭口**：销售及时多发带价值消息是正确积极跟进，**不要**写成空催轰炸；完全不跟或跟太少才写入缺失。
- 判卡点前先问：是客户真因，还是把 SOP 正确步骤误判成了客服问题？
