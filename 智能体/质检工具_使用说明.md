# 聊天记录质检工具 · 使用说明

把每天的聊天记录表格丢进去，自动按销售标准逐通质检，输出一份 Excel 质检报告。
质检标准在 `prompts/full_qc.md`（重点档）与 `prompts/lite_qc.md`（轻量档）；`历史提示词_已废弃.md` 仅为历史存档，程序不会加载。

---

## 一、它做什么

输入：一个或多个 `会话聊天记录-*.xlsx`（表头如 `会话ID / 联系人 / 接待成员 / 社媒渠道 / 会话消息`，列名会自动识别，叫 `会话消息内容` 也认）

处理：
1. 自动拆通、合并续行（`会话ID` 为空的行接到上一通）、去重
2. **按客户发言条数分三档处理**（见下表），减少无效 API 调用、提速
3. 汇总成 `质检报告_年月日_时分秒.xlsx`

### 分级质检（核心提速机制）

| 档位 | 条件（客户发言条数） | 处理 | 报告内容 |
|------|----------------------|------|----------|
| **跳过** | 少于 4 条 | 不调 API | 写入「低意向客户」表：发言数、快速筛选、筛选说明、各次发言时间等 |
| **轻量** | 4 ~ 6 条 | 调 API（短标准） | 结果标签、是否合格、风险等级、红线、主要问题、改进建议、下一步跟进动作 |
| **重点** | 大于 6 条 | 调 API（完整标准） | 完整结构化质检：客户阶段、客户主要顾虑、心理轨迹、对症率、各类顾虑、沉默与激活、逼单跟单等（合格/不合格 + 风险等级，不打分） |

报告按客户发言数拆成两个工作表：**优质客户分析**（≥4 条，含 API 质检结果）与 **低意向客户**（<4 条，规则筛选）。优质客户表有 **`质检档位`** 列（轻量/重点），方便筛选。

优质客户表主要列：质检档位、结果标签、客户阶段、客户主要顾虑、是否合格、风险等级、是否需要人工复核、红线、对症率、疑问及时率、承接评价、顾虑化解（效果/信任/价格/拖延/COD）、客户心理轨迹、沉默与激活、逼单跟单、主要问题、改进建议、下一步跟进动作、原始对话。本工具只做结构化质检判断，不输出任何分数。

## 二、第一步：拿一个 DeepSeek API Key（约 2 分钟）

1. 打开 https://platform.deepseek.com → 注册/登录（支持手机号）
2. 左侧菜单「API Keys」→「创建 API Key」→ 复制（形如 `sk-xxxxxxxx`，**只显示一次，存好**）
3. 「充值」里充几块钱即可（质检一通约几厘钱，几百通也就几块钱）

> 想用别家也行：通义千问填 `https://dashscope.aliyuncs.com/compatible-mode/v1` + 模型 `qwen-plus`；
> OpenAI 填 `https://api.openai.com/v1` + 模型 `gpt-4o-mini`。都是 OpenAI 兼容接口，填法一样。

## 三、第二步：运行

### 方式 A：直接用 Python 跑（你电脑已装 Python 3.11）

```powershell
cd d:\0221\工具库\智能体
pip install -r requirements.txt
python 聊天质检工具.py
```

### 方式 B：打包成 exe，发给同事双击用（无需装环境）

```powershell
cd d:\0221\工具库\智能体
pip install pyinstaller
pyinstaller -F -w --name 聊天质检工具 --add-data "prompts;prompts" 聊天质检工具.py
```

完成后将 `dist\聊天质检工具.exe` 与 **`prompts` 文件夹** 放在同一目录分发（便于后续改质检标准）。若使用 `--add-data`，程序也会从打包资源中读取 prompt 作为备用。

> 注意：`qc_config.json`（存 API Key）会生成在程序所在目录。打包发给别人时不要把含真实 Key 的 `qc_config.json` 一起发出去。

### 维护质检标准（Prompt）

质检规则不再写在 Python 里，而在同目录 `prompts/` 下：

| 文件 | 用途 |
|------|------|
| `prompts/full_qc.md` | 重点档（客户发言 > 6 条）完整质检 |
| `prompts/lite_qc.md` | 轻量档（4~6 条）快速质检 |

修改 md 文件后**重启程序**即可生效（程序内有缓存，同一次运行中不会热更新）。

## 五、操作界面

1. **接口设置**：
   - Base URL：`https://api.deepseek.com`（默认已填）
   - 模型：`deepseek-v4-pro`（默认已填，质量更好；想更快可改 `deepseek-chat`）
   - API Key：粘贴你的 `sk-...`
   - 并发数：8（默认；网络好且不限流可试 10~12，出现 429 就降回 6~8）
   - 点「保存配置」（下次自动记住，存在同目录 `qc_config.json`，**此文件含密钥，勿提交 Git**）

> 首次使用：复制同目录 `qc_config.example.json` 为 `qc_config.json`，填入 API Key 后保存。若缺少配置文件，程序启动时会弹窗提示。
2. 点「测试连接」，看到 ✅ 即正常
3. 点「选择 xlsx 表格并开始质检」→ 多选当天的表格 → 等待
4. 跑完自动打开生成的 `质检报告_*.xlsx`

## 六、怎么看报告

报告有两个工作表，先看 **优质客户分析**，低意向单独在 **低意向客户** 表。

- **先按「质检档位」筛 `重点`**：值得复盘的长对话；客户发言 <4 条的已在另一张表
- **再看「是否合格」「风险等级」「是否需要人工复核」「红线」**：不合格 / 高风险 / 需复核 / 命中红线的单元格会自动标红，优先复盘
- **客户阶段 + 客户主要顾虑**：快速定位客户当前所处环节与核心顾虑
- **对症率、疑问及时率、顾虑列**（重点档）：看应答是否对症、顾虑是否按序化解
- **主要问题 + 改进建议 + 下一步跟进动作**：直接给到客服，附带"应该这样回"的话术与下一步动作
- 想看原文核对，「原始对话」列是完整对话

**低意向客户** 表看「快速筛选」「筛选说明」：如客服首次回复超过 10 分钟会标红。

建议用 Excel「筛选」：`质检档位 = 重点` 且（`是否合格 = 不合格` 或 `风险等级 = 高`），先处理最差会话。

## 七、常见问题

| 现象 | 原因 / 解决 |
|------|------|
| 测试连接 ❌ 401 | API Key 错或没充值 |
| ❌ HTTP 429 | 并发太高/限流，把并发数调小到 6 或更低 |
| 某几通报 `_错误` | 该通太长被超时，可重跑或忽略；程序已自动截断超长对话 |
| 报告打不开 | 文件正被占用，关掉旧报告再跑 |
| 选了表却提示「没有可质检的数据」 | 选错文件了，该表没有「会话消息/会话ID」列；选聊天记录导出表 |
| 想调整分档阈值 | 改 `qc_core.py` 顶部 `TIER_SKIP_MAX`（默认4）、`TIER_LIGHT_MAX`（默认6） |
| 想改质检标准 | 改 `prompts/full_qc.md`（重点档）或 `prompts/lite_qc.md`（轻量档），重启程序 |

## 八、成本参考

DeepSeek 按量计费，一通会话（含 system prompt）约 0.003–0.01 元。
启用分级后，短对话直接跳过不花钱，一天 200 通里通常只有几十通走 API，成本和耗时都更低。
按上次样本（约 6 成是低意向短对话），200 通耗时可从约 30 分钟降到约 10~15 分钟。
