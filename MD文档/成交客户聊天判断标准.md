# 成交客户聊天记录判断标准（v1.0）

> **⚠️ 评分体系已废弃（历史存档）**：本文「聊天质检评分表（100 分）/各维度得分/总分」等打分内容**已停用**。
> 当前线上质检只做**结构化判断、不输出分数**，实际生效标准见 `智能体/prompts/full_qc.md` 与 `prompts/lite_qc.md`。
> 本文其余流程/红线/顾虑分析仍可作复盘参考，但**评分相关章节按已废弃处理**。

> **用途**：上传客户聊天记录后，判断「聊得对不对」「离成交还有多远」「顾虑是否被正确化解」。
> **依据**：`培训_SOP_4.0_整理版.md`、`大力丸销售标准化思维和回复.docx`、**872 条已成交会话** 实证分析。
> **数据范围**：`聊天记录/` 文件夹内 6 个 Excel 导出，合并续行并按会话ID去重后共 872 条成交会话。
> **不适用**：访客标签与聊天标记一致性校验（属另一套逻辑，本文不涉及）。

---

## 一、文档目的

1. 从 **已成交客户** 反推：什么流程、什么动作更容易成交。
2. 把培训 SOP + 话术文档的要求，落成 **可检查、可打分** 的清单。
3. 供人工复盘、智能体质检、新人对照学习使用。

### 客户顾虑出现频率（客户原话关键词统计）

| 顾虑类型 | 占成交会话比例 | 对应培训心理标签 |
|----------|----------------|------------------|

## 三、成交客户的共性特征

### 3.1 人群画像

- **性别**：男性
- **年龄**：多为广告进线 `【💪40+】`～`【💪70+】` 分段，或主动报年龄（如 "63"）
- **市场**：以澳大利亚 Messenger 进线为主
- **需求**：阳痿、早泄、尺寸、前列腺、肾虚、精力不足（与 SOP ①–⑤ 一致）
- **互动深度**：深度聊天型，中位 **70 条** 客户发言，远超「优2」的 6 条门槛

### 3.2 行为共性

1. **愿意吐露具体问题**（时长、硬度、尺寸 cm、用药史、婚姻压力等）
2. **会主动问价格**，或接受方案后进入付款讨论
3. **经历过至少一轮异议**（效果/信任/价格三者中至少一类）
4. **最终出现行动信号**：给地址、收链接、付定金、发收据

### 3.3 成交 ≠ 完美执行 SOP

实证显示并非所有成交都走完 Word ①–⑨ 每一小节：

- 仅 **45%** 有完整 Tina 人设话术
- 仅 **28%** 有 Google Drive 链接
- 英文「档案」话术在文本中几乎不出现

**因此判断标准必须区分两层：**

- **硬性必达（商业化闭环）**：症状厘清 → 方案/报价 → 异议处理 → 付款动作
- **软性建议（提升转化率）**：人设、问诊深度、案例链接、档案、逼单全套

## 四、标准成交流程（黄金路径）

顺序与 872 条成交会话的中位路径及 SOP 4.0 三步结构一致：

```
进线钩子 → 人工接手 → 信任建立 → 症状厘清 → 方案价值 → A/B报价
    → 异议化解（效果→信任→价格→拖延） → 行动促成 → 付款确认
```

### 阶段 0：进线

| 检查项 | 合格 | 不合格 |
|--------|------|--------|
| 进线识别 | 客户有广告标签/emoji/年龄信号或主动打招呼 | 客服无视首句直接甩长文 |
| 非人工消息 | 「自动化」下班回复忽略，不计入考核 | — |

### 阶段 1：信任建立（SOP 第一步 · Word ①）

| 检查项 | 合格 | 成交样本参考 |
|--------|------|--------------|
| 人设 | Dr. Tina / 同仁堂 / 8年 / 30万案例 / 澳洲 | ~45% 有完整版；至少有专业身份+资质/媒体 |
| 媒体 | 名片、案例图、学历、医师证 | 常见连发 4 张媒体后接文字人设 |
| 收尾 | **必须以问号/引导结尾** | SOP 铁律：永远留问号 |

### 阶段 2：症状厘清（SOP 第二步 · Word ②–④）

| 检查项 | 合格 | 不合格 |
|--------|------|--------|
| 症状菜单 | 引导回复 1–5 编号或等价五类症状 | 未定位症状直接报价 |
| 得知症状 | ≤2 / >2 症状有不同安抚话术 | 忽视客户已说症状 |
| 问诊深度 | 至少覆盖病程、具体时长/硬度/尺寸、用药史或慢性病中 ≥2 项 | 只聊 1 轮就推产品 |
| 表达逻辑 | 用西医/药理逻辑解释（血流、激素、神经、5-HT） | 堆中文术语客户看不懂 |
| 具象化 | 封闭式提问：「3分钟还是5分钟？」「硬度图哪一档？」 | 接受模糊回答不追问 |

### 阶段 3：方案与价值（SOP 第三步 · Word ⑤）

| 检查项 | 合格 |
|--------|------|
| 配方故事 | 万历/大力丸/右归丸+五子衍宗丸（可简化） |
| 成分 | 13 味药材 / 高倍浓缩 / 定制非流水药 |
| 优势 | 纯天然、无副作用、非伟哥式暂时刺激、一对一跟踪 |
| 发送方式 | **分条发送**，非一大段长文 |

### 阶段 4：报价（Word ⑥ · SOP 第四天）

| 检查项 | 合格 | 不合格 |
|--------|------|--------|
| 双方案 | A：400 AUD（原价800）/ B：300 AUD（原价600） | 只报一个价、价格不符 |
| 对比 | 13药100倍 vs 8药50倍；症状重推 A 留 B 口 | 强推低价、贬低 A |
| 推荐逻辑 | 按症状/信任主推 A，但不说死 | 预算不足仍硬逼 A 全款 |
| 前置条件 | 报价前已完成症状与价值铺垫（高意向主动问价除外） | 无问诊直接甩价 |

### 阶段 5：收信息与档案（Word ⑦⑧）

| 检查项 | 合格 |
|--------|------|
| 信息模板 | 姓名、电话、邮箱、精确地址（门牌）、邮编 |
| 邮箱说明 | 用于物流追踪 |
| 档案 | Dali Pill 档案 + 编号（理想有） |
| 澳洲经验 | 客户主动给地址是强成交信号，应在报价/逼单后推进 |

### 阶段 6：逼单与跟单（Word ⑨ · SOP 重1/重2）

| 客户状态 | 定义 | 客服动作 |
|----------|------|----------|
| **重2** | 报完价未选方案 | 先让用户开口，弄清效果 vs 价格顾虑，再跟单 |
| **重1** | 选方案/给地址/发链接/垫资 | 分析无法当天付款原因，走 福利→定金→垫资 阶梯 |
| 逼单工具 | 24h优惠、补肾壮阳贴赠品、**50 AUD 定金**、100 AUD/7天体验 | 环境建全再逼，不空逼 |
| 已读不回 | 先解决当下顾虑，不是继续堆产品 | 放大痛点 / 问是否担心收货或效果 |
| 未读未回 | 2小时一条置顶，案例/视频；尝试 WhatsApp | 狂轰滥炸 |

### 阶段 7：付款闭环（Word ⑩）

| 检查项 | 合格 |
|--------|------|
| 支付链接 | `cashier.pricpay.com` 或财务生成链接 |
| 收据 | 要求客户发 receipt / 付款截图 |
| 确认到账 | 排除 P 图、延时到账后再发货 |
| WhatsApp | 重1/一般客户应收 WhatsApp，便于跟单 |
| 转经理 | 成交后报单、改标签、转发货 |

## 五、顾虑化解标准（成交客户实证 + SOP 对照）

### 5.1 A 类 · 不信效果（88.9% 成交客户出现过）

**客户典型表现**
- Does it really work? / I hope yours works
- 报具体尺寸、时长、晨勃、用药无效史
- 要案例、要数据、要前后对比

**正确化解（成交会话常见组合）**
1. 认可顾虑 →「很多人一开始也担心…」
2. 具象案例 → 同城/同症状客户（如 Peter from Adelaide）
3. 机制解释 → 血流、睾酮、神经、5-HT，非玄学
4. Google Drive 视频（可选但有效）
5. 定制方案 → 针对他说的 1-2min、11cm 等逐项回应
6. 不急于逼单，先建立「结果信心」再报价

**错误动作（培训明文禁止）**
- 忽略效果质疑直接报价逼单
- 只重复「我们很好」无案例
- 承诺无法兑现的具体数字

**质检问题**：客户提出效果疑虑后，客服 3 轮内是否给出「认可 + 案例/机制」？

---

### 5.2 B 类 · 不信你 / 怕骗（43.2%）

**客户典型表现**
- Facebook 广告不信、被骗过、要钱才发货
- 要地址、执照、自拍、视频通话
- How can I trust you?

**正确化解**
1. 共情 →「换作是我也会担心」
2. 资质 → 医师编号、官网可查、Tina 身份
3. 证据栈 → Drive 视频、FB 主页、公司地址（Adelaide / Macquarie Park）
4. PayPal 退款机制（适用时）→ 强调敢用 PayPal 说明真实
5. 定制药不可退 → 解释为何先付款、为何不能 COD
6. WhatsApp 移交 → 降低「广告号骗子」感受

**错误动作**
- 疯狂证明公司、过度防御
- 与客户争论
- 辱骂或嘲讽（黑粉也禁止辱骂）

**质检问题**：出现 scam/trust 关键词后，是否出现「共情 + 至少一种权威证明」？

---

### 5.3 C 类 · 价格 / 预算（70.8%）

**客户典型表现**
- How much in AUD? / no budget / too expensive
- 主动问价不一定是拒绝，可能是购买前确认

**正确化解**
1. A/B 双方案 → 给客户台阶
2. 价值锚定 → 药材稀缺、定制、一次性永久 vs 伟哥持续花钱
3. 比价 → 医院上万、西药副作用
4. 50 AUD 定金 保留优惠和赠品（21.7% 成交走过）
5. 分期 / 体验装 100 AUD/7天（Word 重2 台阶）
6. 不一味降价

**错误动作**
- 客户说贵就立刻打折到底
- 无价值铺垫只谈钱

**质检问题**：报价后客户喊贵，是否走「价值重申 → 定金/分期」而非直接放弃？

---

### 5.4 D 类 · 拖延 / 商量（30.0%）

**客户典型表现**
- I'll think about it / talk to wife / wait / try something else first

**正确化解**
1. 支持决定 + 共情（和妻子商量话术：妻子未必支持男科消费）
2. 得失对比 → 今天付能得到什么、拖下去失去什么
3. 时间节点 → 24h 优惠、23:00 截止、药材配额
4. 低压力跟进 → 「先试别的」型，nurture 后回来付款
5. WhatsApp 保持联系

**错误动作**
- 持续高压逼单
- 客户说考虑就消失

**质检问题**：客户说 consider 后，是否有约定时间点的跟进或痛点再激活？

---

### 5.5 货到付款（6.2%）

**正确化解**
- 明确不支持 COD
- 原因：定制药、不可二次销售、本地快递员收不到钱的风险经验
- 替代：PayPal 保障 / 定金降低风险 / 物流单号

**错误动作**
- 答应货到付款
- 不解释原因只拒绝

---

### 5.6 E 类 · 情绪陪伴型（培训警示）

成交样本中部分客户高互动、聊生活，需注意：

- **真需求**：会主动回到产品、方案、付款
- **假需求**：只情绪不推进 → 应控制时间，定期拉回症状

**质检**：连续 10 轮无产品话题 → 标记「情绪偏离」，需引导回方案。

## 六、客户心理实时识别（逐条发言分析）

> 目标：不只看「客户属于哪类」，而是 **逐条** 读客户发言，标出 **当下心理**，作为评判销售应答是否对症的前提。
> 原则（SOP 第一性）：**网销第一性永远是「用户当下说了什么」**；先判心理，再谈话术。

### 6.1 单条发言的心理标注维度

对每一条客户发言，按以下维度打标：

| 维度 | 取值 | 识别线索 |
|------|------|----------|
| **意图类型** | 提问 / 陈述症状 / 表达顾虑 / 还价 / 同意推进 / 闲聊 / 拒绝 / 沉默信号 | 句式、问号、关键词 |
| **情绪温度** | 热（想买）/ 温（观望）/ 冷（防备/敷衍）| 回复长度、主动性、是否带情绪词 |
| **心理标签** | A效果 / B信任 / C价格 / D拖延 / E情绪 / F观望 | 见第五节六类 |
| **推进信号** | 强（问价/给信息/问怎么付）/ 中（追问细节）/ 弱（嗯、ok、再看看）| 是否朝成交方向移动 |
| **隐藏潜台词** | 真实诉求 vs 表面说法 | 见下表 |

### 6.2 「表面话 → 潜台词」对照表（成交样本高频）

| 客户表面说法 | 大概率潜台词 | 当下心理 | 销售应切入 |
|--------------|--------------|----------|------------|
| Does it really work? | 我被骗过/不敢信 | A+B | 认可+案例+机制 |
| How much in AUD? | 我在认真考虑买 | C（偏热）| 给 A/B 双方案，别只报一个 |
| Too expensive / no budget | 想买但钱紧 | C | 定金/分期/单次成本，不是放弃 |
| I'll think about it | 还有没说出口的顾虑 | D | 主动问「是效果还是价格」 |
| Talk to my wife | 怕家里反对/找台阶 | D | 共情+得失+把妻子拉同阵营 |
| Can I pay on delivery? | 怕收不到货 或 没钱 | B/C | 解释不可COD+证明真实 |
| 已读不回 / 只回"ok" | 顾虑未解决，热度下降 | 任意 | 回到上一个未解决的点 |
| 只聊生活、不谈产品 | 缺陪伴，可能假需求 | E | 共情但定期拉回方案 |
| 报具体尺寸/时长 | 高意向、愿配合 | 热 | 顺势问诊→方案 |

### 6.3 心理变化轨迹（会话级）

一条完整会话应能画出心理曲线，质检关注 **转折点**：

- **升温点**：客户从「冷/防备」转「主动问价/给信息」——记录是哪句销售话促成。
- **降温点**：客户从「热」转「已读不回/还价/质疑」——记录销售上一句是否踩雷（如无铺垫硬报价、忽略问题）。
- **判定**：成交会话通常有 ≥1 个清晰升温点；若出现降温点，应有销售 **及时挽回** 动作。

## 七、销售应答质量评估（是否对症、及时、承接合理）

> 这一节是核心：判断 **销售的话能不能合理解决客户当下心理**、**疑问是否及时解答**、**承接客户的话是否合理**。

### 7.1 维度一：应答是否「对症」（解决客户心理）

逐个「客户顾虑 → 销售应答」配对，判定是否对症：

| 判定 | 标准 |
|------|------|
| ✅ 对症 | 销售识别出客户当下心理标签，并用第五节对应化解组合回应 |
| ⚠️ 偏题 | 答了，但没针对真实潜台词（如客户问效果，只回价格）|
| ❌ 错配 | 用错方向（如 B 信任顾虑却疯狂打折，C 价格顾虑却继续讲机制）|
| ⛔ 忽略 | 客户明确顾虑，销售完全没回应（违反第一性原则）|

**对症率** = 对症应答数 / 客户顾虑总数。成交样本健康值参考 **≥80%**。

### 7.2 维度二：疑问是否及时解答

| 检查项 | 合格标准 | 不合格 |
|--------|----------|--------|
| 直接问题响应 | 客户提出明确问题（价格、地址、成分、能否治…），**下一两条**销售消息内必须正面回应 | 绕开、跳过、留到很久以后 |
| 不拖延关键问题 | 价格、是否能治、怎么付款等核心问题不得回避 | 用「稍后说」「先做个检查」长期搁置 |
| 多问题完整度 | 客户一次多个问题，**逐条都答**，按 SOP「从下往上」顺序 | 只挑一个答、漏答 |
| 未答清单 | 会话结束仍有客户问题未答 → 全部列出 | — |

**及时率** = 被及时解答的问题数 / 客户提问总数。

### 7.3 维度三：承接客户的话是否合理（衔接质量）

> 「承接」= 销售是否接住客户上一句，再自然带向下一步；这是 SOP「永远留问号 / 顺着客户往下聊」的落地。

| 承接类型 | 说明 | 评价 |
|----------|------|------|
| ✅ 接住+推进 | 先回应客户上一句，再抛出下一个引导问题 | 优 |
| ✅ 接住+共情 | 情绪类先共情再处理 | 优 |
| ⚠️ 只接不推 | 答了但没引导，对话停滞、要客户自己找话 | 中 |
| ⚠️ 只推不接 | 不理客户上一句，自顾自发下一段话术 | 差 |
| ❌ 答非所问 | 接的点和客户说的不是一回事 | 差 |
| ❌ 话术轰炸 | 连发多段模板，无视客户节奏（SOP 禁止：别一次发一大段）| 差 |

**承接检查要点**
- 每条销售消息是否 **回扣客户最近一句的关键词/情绪**？
- 关键节点是否 **以问号结尾**，把球踢回给客户？
- 是否避免「发完人设/案例就没下文」（SOP 明确禁止）？

### 7.4 维度四：节奏与分寸

| 检查项 | 合格 | 不合格 |
|--------|------|--------|
| 信息密度 | 分条发送，一条解决一个顾虑 | 长篇大论 walls of text |
| 推进速度 | 与客户温度匹配（热可快、冷需慢） | 客户冷却仍硬逼 |
| 销售感控制 | 先做「靠谱顾问」再卖货（B 类尤其）| 一上来就推销逼单 |

## 八、客户沉默的二次激活与触达标准

> 针对「客户不回消息」，判断 **二次激活/触达是否到位**。区分「已读不回」与「未读未回」，二者策略不同（SOP 第五天明确）。

### 8.1 先判沉默类型与心理

| 类型 | 心理推断 | 关键动作方向 |
|------|----------|--------------|
| **已读不回** | 仍有未解决顾虑（价格/效果/信任）| **先解决顾虑**，不是继续逼单 |
| **未读未回** | 外界干扰、精力转移，需求仍在 | **保持置顶**，等待+换渠道 |
| 报价前沉默 | 某阶段心理卡住 | 回看上下文定位阶段，用「起因-经过-结果」一激开口 |
| 报价后沉默 | 最初质疑点仍在 | 第一性不是逼单，是解决疑虑 |

### 8.2 二次激活动作检查表

| 检查项 | 合格标准 | 不合格 |
|--------|----------|--------|
| 是否有激活动作 | 客户停聊后销售 **主动触达**，未放任流失（SOP：不挂0、敬畏流量）| 客户不回就再无下文 |
| 已读不回处理 | 先回扣 **上一个未解决顾虑** / 放大痛点 / 主动问「是否担心收货或效果」 | 无视顾虑继续发方案、硬逼 |
| 未读未回处理 | 尝试 **加 WhatsApp**；无电话则索要；**约每 2 小时一条**，保持置顶 | 频繁轰炸 / 完全不跟 |
| 触达内容质量 | 发 **案例 / 视频链接 / 治疗场景 / 痛点具象**，每次有新价值 | 反复「在吗」「考虑得怎样」 |
| 触达频率 | 未读期不过密；已读期结合顾虑节奏 | 短时间连发十几条施压 |
| 换渠道 | Messenger 沉默后尝试 WhatsApp 移交（成交样本 62% 提及）| 死守一个渠道直到流失 |
| 收尾通牒 | 第一天有「今天不订赠品/优惠失效」的合理收口 | 无任何时间锚点 |

### 8.3 触达质量评分

- **激活及时性**：客户沉默到首次触达的间隔是否合理（不过快显得逼，不过久致流失）。
- **激活有效性**：触达后客户是否重新开口；成交样本里「跟单中回复」的客户往往是高需求，需重点分析其之前为何沉默。
- **轮次记录**：第几天、第几次触达、用了什么内容、是否按 重1 七天 / 重2 三天 节奏。

## 九、逼单与跟单手段适用性评估

> 判断 **逼单/跟单手段是否用得合适**：对的客户、对的时机、对的力度、环境是否建全。

### 9.1 前提：先确认能不能逼

| 前置条件 | 说明 |
|----------|------|
| 过程是否走完 | **报价前不逼单**；过程没走完先补过程（SOP：过程好=结果好）|
| 顾虑是否已解 | 信任/效果顾虑未解时逼单 = 无效动作甚至反效果 |
| 标签是否正确 | 先分清 **重1（已选方案）/ 重2（未选方案）**，策略不同 |

### 9.2 重2（未选方案）跟单评估

| 检查项 | 合格 | 不合格 |
|--------|------|--------|
| 第一性 | 先让客户 **开口**，问明顾虑是 **效果还是价格** | 直接发逼单话术 |
| 已读不回 | 先发导语/日常切入，再发跟单内容 | 上来就甩跟单模板 |
| 节奏 | 三天跟单，内容多时分次发 | 一次性轰炸 |
| 降级规则 | 超 3 天降标签到优2，进复洗，不死缠 | 无限跟单/直接放弃 |

### 9.3 重1（已选方案）逼单评估

逼单工具应 **按阶梯递进**，且每一步 **环境建全** 后再用：

| 阶梯 | 手段 | 适用前提 | 用错的表现 |
|------|------|----------|------------|
| 1 | **福利赠品**（补肾壮阳贴/24h 优惠）| 客户想买、临门一脚 | 没共情就直接甩优惠 |
| 2 | **定金保留**（50 AUD 起）| 想买但当天付不了全款 | 没说明稀缺/申请就要定金 |
| 3 | **情感垫资** | 真有意向、前两步无效 | 未给交换条件、用力过猛致怀疑 |

**逼单合理性检查表**

| 检查项 | 合格 | 不合格 |
|--------|------|--------|
| 共情先行 | 逼单前先共情客户处境 | 无共情硬逼 |
| 稀缺/限时有解释 | 优惠/赠品/名额说明为何稀缺、为何限时 | 空喊「今天最后一天」无理由 |
| 给台阶 | 全款→定金→垫资逐级降门槛 | 只盯全款，客户没钱就放弃 |
| 力度匹配 | 与客户温度/类型匹配（D 拖延给时间节点，C 价格给分期）| 对观望客户过度施压 |
| 垫资分寸 | 有交换条件（转介绍）、明确金额与归属 | 单方面付出、引客户起疑 |
| 不空逼单 | 每次逼单都带 **新理由/新价值** | 反复同一句催 |
| 跟单节奏 | 重1 七天逐日、内容对应天数 | 无计划乱跟或断更 |
| 降级处理 | 超 7 天降重点、保留重1、转洗客户 | 无限纠缠或直接拉黑 |

### 9.4 市场差异（影响逼单判定）

- **澳大利亚**：选方案 ≈ 成功 60%，给地址 ≈ 成功 80%；可适度推进给地址。
- **南非**：对给地址/选方案不排斥，习惯超前消费；推进可更快。
- 质检时按 `社媒渠道` 与客户语言判断市场，套用对应预期。

## 十五、真实对话取证（逐句精读 10 通成交全文）

> 以下均为 `聊天记录/` 中真实成交对话原句（已脱敏到名），用于校准标准、给新人当范例。
> 关键提醒：**真实成交里也有错误动作，靠后续挽回**——评判时要看「整体是否挽回」，不是单句一票否决。

### 15.1 信任异议（真实问答对）

**Marc Shaddock（叶兴儒 / hqh1001）**

| 客户原句 | 客服应答 | 评价 |
|----------|----------|------|
| "I think you are trying to sell me something… Very dishonest" | "Why would you think that? I'm a real doctor." | ❌ 反面：第一反应防御、反问质疑客户 |
| （紧接上句）| "I understand why you might feel this way. If my words gave you that impression, I'm sorry… Whether to proceed is entirely your decision; I won't pressure you." | ✅ 挽回：共情+降压 |
| "I am a bit sceptical of anything sold on Facebook… 90% of the stories are fake" | "I completely understand… The difference is I'm not selling a generic product… I prefer to communicate directly with you rather than just sending a product link." | ✅ 对症：区隔「广告骗子」 |
| "How can I trust what you are saying… can I get my money back?" | 亮证：AHPRA 注册号 `CMR0002510454` + "all practitioners are subject to strict consumer protection laws… you can file a complaint" | ✅ 权威背书 |
| "I can't find anything about your business, do you have a Website?" | 给官网 + "SYDNEY CHINESE MEDICINE CENTER, ABN 71 109 163 395" | ✅ 可验证透明 |

**Biju Chacko（叶兴儒）**：客户 "big trust issue with online payment, many times I have been scammed" → 客服先关系预热（Easter 问候、"I've been thinking about you"）再 "all my information is open and transparent, you can verify it any time" → 成交。

**Darren Goodman（雷岩）**：客户 "I've been scammed before, don't like giving card details online" → 客服立即迁就 "bank transfer? Sure, no problem at all" → 用**支付方式灵活性**拆掉信任墙。

> **校准点**：信任异议成交的共同公式 = **共情 → 可验证证据（AHPRA/ABN/官网/PayPal）→ 迁就支付方式**。防御性反问是 missteps，应扣分但若后续挽回则不致命。

### 15.2 价格 / 垫资博弈（真实）

**Marc Shaddock**

- 客户："The only way my wife would let me do this is if I could pay you $100 now and then pay the other $300 if it works"
- 客服（不接受 100，给台阶到付一半发一半）："If you could pay $200 upfront, I can mail you half of the medication… I really can't ship for just $100."
- 结果：客户改口 "Plan A" + 约定 6pm 付款。

> **校准点**：面对「先付一点」，正确不是答应也不是拒绝，而是 **反提一个折中（付 200 发一半）**，既降门槛又守底线。文档第九节「定金/垫资阶梯」由此得到真实佐证。

### 15.3 二次激活 / 触达（真实节奏）

**Marc Shaddock** 多次出现：

- 早安破冰："Good morning. Did you sleep well last night?" / "Are you at work right now?"
- 精确时间锚点："I'll be waiting for your results tomorrow at 11 a.m. so I can prepare your herbs in advance."
- 沉默后跟进："Are you busy right now? Did you see the message I sent?"

> **校准点**：真实成交的触达 = **早安/关心式破冰 + 给一个具体时间点 + 准备药材的紧迫感**，而非「在吗/考虑得怎样」。验证第八节触达质量标准。

### 15.4 反面/异常样本（必须纳入判读）

| 客户 | 现象 | 对标准的意义 |
|------|------|--------------|
| **Bahram Haidari** | 成交并收到药，但**售后失联**，客户连发 "Why not replying??" "You are online why not reply??" "if not results we not charge for second treatment" | 成交≠合格；**售后响应**也要质检；触点检测里他**无 payment_link/receipt** 仍成交 |
| **Ryan Hussain** | **8 个标准触点一个都没命中**，靠尊重隐私+轻量跟进成交 | 不能只用「触点齐不齐」判分；**关系型路径**真实存在 |

> **校准点**：评分表（第十一节）必须保留「心理应答 / 承接 / 激活」等过程分，**不能只看流程触点**，否则会误判 Ryan 这类成交为不合格、误判 Bahram 这类为优秀。

### 15.5 由真实数据得到的修正

1. **触点是参考非铁律**：Tina 完整人设仅 45%、symptom menu 64%——缺触点不直接判错，要结合心理应答看。
2. **防御性反问普遍存在**：真实客服也会先反问，标准应「记录 misstep + 看是否挽回」。
3. **售后窗口要纳入**：成交后客户追问效果/物流若失联，应单列「售后响应」扣分项。
4. **支付方式灵活性是成交杠杆**：客户拒卡就给银行转账/PayPal，是高频成交动作，已并入第五节 B 类与第九节。

## 十六、判读总流程（建议执行顺序）

```
1. 解析会话：按时间/说话人拆分，区分 客户 / 客服 / 自动化（自动化不计分）
2. 逐条标客户心理（第六节）→ 画心理轨迹
3. 配对「客户顾虑/问题 → 销售应答」→ 评对症率、及时率、承接质量（第七节）
4. 检测沉默段 → 评二次激活与触达（第八节）
5. 定位逼单/跟单动作 → 评时机、标签、阶梯、力度（第九节）
6. 对照成交原型（第十节）
7. 按评分表（第十一节）打分 + 红线检查（第十二节）
8. 输出报告（第十三节模板）
```
